# Котность Bot 🐱

Telegram-бот по мему **«Котность»**: пользователь создаёт своего кота, даёт ему кличку и прокачивает кота действиями через кнопки.

## Возможности

- Создание кота при `/start` с выбором клички.
- Кнопочные действия:
  - 🍗 Покормить
  - 🖐 Погладить
  - 🛁 Помыть
  - 📊 Профиль
  - 🏆 Лидерборд
- Начисление очков **«Котость»** за действия.
- Топ-10 лидерборд по очкам котости.

## Стек

- TypeScript
- Node.js
- [Telegraf](https://telegraf.js.org/)
- Docker + Docker Compose

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=ваш_токен_telegram_бота
```

Можно использовать `.env.example` как шаблон.

## Локальный запуск

1. Установите зависимости:

```bash
npm install
```

2. Соберите проект:

```bash
npm run build
```

3. Запустите бота:

```bash
npm start
```

Для разработки в watch-режиме:

```bash
npm run dev
```

## Запуск через Docker Compose

1. Убедитесь, что в `.env` задан `BOT_TOKEN`.
2. Запустите:

```bash
docker compose up --build -d
```

3. Логи контейнера:

```bash
docker compose logs -f kotnost-bot
```

4. Остановка:

```bash
docker compose down
```


> Если в логах контейнера есть `ETIMEDOUT` или `EAI_AGAIN` при обращении к `api.telegram.org`,
> это обычно проблема сети/DNS. Бот теперь не падает окончательно: он продолжает
> автоматически переподключаться с задержкой (по умолчанию 10 секунд).

## Структура проекта

- `src/index.ts` — основная логика Telegram-бота.
- `Dockerfile` — контейнеризация приложения.
- `docker-compose.yml` — запуск сервиса через Compose.
- `.env.example` — пример переменных окружения.
