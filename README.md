# Котность Bot 🐱

Telegram-бот по мему **«Котность»**: пользователь создаёт кота, ухаживает за ним через кнопки и прокачивает очки котости.

## Что умеет бот

- Создание кота при `/start` с вводом клички.
- Игровые действия через inline-кнопки:
  - 🍗 Покормить (`+3` котости)
  - 🖐 Погладить (`+2` котости)
  - 🛁 Помыть (`+4` котости)
- Просмотр профиля кота (текущая котость и статистика действий).
- Лидерборд топ-10 по очкам котости.
- Напоминания о неактивности:
  - если пользователь не взаимодействовал с котом **8 часов**, бот отправляет напоминание;
  - за игнор предыдущего напоминания снимается небольшой штраф котости (по умолчанию `-2`);
  - котость **никогда не падает ниже 0**.

## Технологии

- TypeScript
- Node.js
- [Telegraf](https://telegraf.js.org/)
- SQLite (`better-sqlite3`)
- Docker + Docker Compose

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN=ваш_токен_telegram_бота
# опционально, задержка ретрая запуска при сетевых ошибках (мс)
RETRY_DELAY_MS=10000
```

Можно использовать `.env.example` как шаблон.

## Локальный запуск

1. Установите зависимости:

```bash
npm install
```

2. Соберите проект:

```bash
npm run build
```

3. Запустите бота:

```bash
npm start
```

Для разработки в watch-режиме:

```bash
npm run dev
```

## Запуск через Docker Compose

1. Убедитесь, что в `.env` задан `BOT_TOKEN`.
2. Запустите:

```bash
docker compose up --build -d
```

3. Логи контейнера:

```bash
docker compose logs -f kotnost-bot
```

4. Остановка:

```bash
docker compose down
```

> Если в логах есть `ETIMEDOUT` или `EAI_AGAIN` при обращении к `api.telegram.org`,
> это обычно проблема сети/DNS. Бот автоматически переподключается с задержкой.

## Структура проекта

- `src/index.ts` — точка входа и Telegram-обработчики, запуск reminder-цикла.
- `src/catLogic.ts` — логика работы с БД, котостью, активностью и штрафами.
- `src/types.ts` — общие TypeScript-типы.
- `kotnost.sqlite` — локальная SQLite база (создаётся автоматически при запуске).
- `Dockerfile` — контейнеризация приложения.
- `docker-compose.yml` — запуск сервиса через Compose.
- `.env.example` — пример переменных окружения.
